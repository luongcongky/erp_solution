# Dashboard Guide

## 1. Overview
**Purpose**: The Dashboard serves as the central hub for the ERP system, providing a high-level overview of business performance, key metrics, and recent activities. It allows users to quickly assess the health of the business and navigate to critical areas.
**User Roles**: All users (Admin, Sales, Inventory, etc.) have access, but data visibility may vary based on permissions.

## 2. Business Process
### Input Data
- **Sales Data**: Aggregated from Sales Orders and Invoices.
- **User Data**: Active user counts from the User Management module.
- **Inventory Data**: Stock levels and alerts from the Inventory module.
- **System Activity**: Logs of recent user actions.

### Output Data
- **Visual Insights**: Charts and graphs representing trends.
- **Alerts**: Notifications for pending issues or required actions.

### Process Flow
1.  **Login**: User logs into the system.
2.  **View Overview**: User lands on the Dashboard to see daily/monthly stats.
3.  **Analyze Trends**: User checks revenue and category charts.
4.  **Take Action**: User clicks on "Quick Actions" or specific stats to navigate to detailed screens (e.g., clicking "New Orders" goes to Sales Orders).

## 3. Screen Details
### Key Metrics (Stat Cards)
| Metric | Description | Source |
|--------|-------------|--------|
| **Total Sales** | Total revenue for the selected period. | Sales Module |
| **Active Users** | Number of users currently active or logged in recently. | Core Module |
| **New Orders** | Count of orders received in the selected period. | Sales Module |
| **Pending Issues** | Count of items requiring attention (e.g., low stock, pending approvals). | Various Modules |

### Charts
- **Revenue Over Time**: Area chart showing sales trends over the last 6 months.
- **Sales by Category**: Bar chart breaking down sales by product category.

### Recent Activity
- Displays a chronological list of important system events (e.g., "John Doe created order #123").
